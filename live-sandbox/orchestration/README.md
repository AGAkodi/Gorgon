# Session Orchestration (`live-sandbox/orchestration`)

Handles the containerized lifecycle and environment isolation for
interactive browser sessions in the Layer 2 Interactive Live Sandbox.

## Modules

### `session.py`
`Session` class wrapping a disposable Playwright Chromium browser context:

- **`async start(target_url)`** — launches headless Chromium, creates an
  incognito context (no persistent `user_data_dir`, no cookies, no cache),
  installs the network egress filter, and navigates to the target URL.
- **`async stop()`** — clears cookies, closes context + browser, stops
  Playwright. No state persists between sessions.
- **Context manager** — `async with Session() as s:` ensures teardown even
  on exceptions.
- **Session ID** — each session gets a UUID4 for tracking.

### `network_filter.py`
`NetworkFilter` class using Playwright's `page.route("**/*", ...)` API:

- Allows requests to the target domain, its subdomains, and a default
  allowlist of common CDN/font origins.
- Blocks everything else with `route.abort("blockedbyclient")`.
- Logs all blocked URLs for debugging.
- `data:` and `blob:` URIs (inline page assets) are always allowed.

This is application-level egress filtering, not a container-level firewall
(that's the Hardening sub-phase).

## Dependency on `sandbox/`

This module maintains the isolated execution environment where page
interaction happens, but depends on `sandbox/` (Layer 1 simulation) to
process transaction payloads generated by the page — that routing happens
in `interceptor/`, not here.

## Tests

```sh
py -3.14 live-sandbox/orchestration/test_session.py
```

Verifies:
- Session lifecycle (launch → load → teardown)
- Session ID is valid UUID4
- Network filter blocks off-domain requests
- Network filter allows target domain + CDNs
- Multiple sessions are independent
